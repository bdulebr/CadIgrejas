Data: 22 de Junho de 2026 - 10:58

**Erro:** `AttributeError: 'Membro' object has no attribute 'departamento_responsavel'` gerando falhas 500 no `sysadmin` e `perfil`.
**Motivo:** As tags de verificação de permissões do sistema (`permissoes_tags.py`) estavam utilizando um atributo obsoleto (`user.departamento_responsavel`) que não reflete a estrutura atual do banco de dados para os membros.
**Solução Aplicada:** Refatoramos a lógica de validação. O sistema de permissões agora confere corretamente se o usuário lidera departamentos com a permissão específica ativa, usando a função `hasattr(user, 'departamentos_liderados')` e filtrando com `departamento__in=user.departamentos_liderados.all()`.
**Regra de Auto-Correção:** Nunca presuma ou force o uso do campo singular `departamento_responsavel` em objetos `Membro` para validação de privilégios. A relação oficial do sistema para autorizações em grupo reside nas tabelas de junção ManyToMany (ex: `departamentos_liderados`). O AI Daemon deve sinalizar qualquer tentativa de acesso a este atributo singular extinto e bloquear PRs/Edições de código.
