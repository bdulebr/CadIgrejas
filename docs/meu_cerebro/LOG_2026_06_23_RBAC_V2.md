# RBAC 2.0 - Controle de Permissões Avançado

**Data:** 23/06/2026
**Módulo:** `permissoes`

## Mudanças Arquiteturais Realizadas

### 1. Novo Módulo Oficial: Recursos Humanos (rh)
O módulo de RH agora possui sua própria permissão no motor do sistema (`ModuloSistema`), o que permitiu desvincular o Dossiê de RH da permissão genérica de "Editar Membros".
- **Visualização de Dossiê:** A view `rh_dossie_membro` foi blindada com uma verificação manual rigorosa baseada na função `obter_escopo_acesso(request.user, 'rh')`. Apenas Super Admins, voluntários com permissão global de RH ou líderes diretos do membro avaliado podem acessar o dossiê.

### 2. Painel de Permissões Expandido
A tela de Gestão de Permissões (`permissoes/dashboard.html`) passou por uma expansão maciça de funcionalidades (RBAC 2.0).
- **Perfis de Acesso (Roles):** Foi ativada a interface visual para criar, excluir e gerenciar a matriz de risco de Grupos/Perfis (`PerfilAcesso`). Também é possível adicionar e remover múltiplos membros de um grupo rapidamente.
- **Escopo de Acesso (RLS - Row Level Security):** As permissões agora renderizam campos de *dropdown* para definir se a permissão concedida tem efeito **Global**, apenas sobre o **Departamento** do usuário, ou **Própria** (Estrita).
- **Expiração:** Adição do suporte via UI à `data_expiracao`, permitindo que os acessos ao sistema se encerrem automaticamente em data futura sem intervenção humana.

### 3. Melhorias de Performance
- A view `painel_permissoes` foi refatorada para utilizar estruturas de dicionários pré-processados, evitando N+1 queries complexas que existiam ao renderizar milhares de caixas de seleção na tabela de usuários.

## Conclusão da IA de Autoaprendizado
A proteção de dados sensíveis na Igreja (como os do módulo RH e LGPD) demanda granularidade. O uso do decorador genérico ou do check de "is_super_admin" puro não escala. Ao lidar com novas features confidenciais, deve-se sempre consultar a matriz do RBAC via `obter_escopo_acesso` e assegurar cruzamento de chaves na visualização dos dados (ex: confirmar se o usuário e a entidade visualizada pertencem ao mesmo departamento caso o escopo seja `departamento`).
